#  Shaurma Tracker - Rostov-on-Don

Веб-приложение для мониторинга и оценки точек продажи шаурмы в Ростове-на-Дону. Пользователи могут находить лучшие места, делиться отзывами и добавлять новые локации на карту города.

##  Основные возможности

- **Система аутентификации:** Полный цикл регистрации и авторизации пользователей (на базе Devise).
- **Трекер точек:** Просмотр списка шаурмечных Ростова-на-Дону с адресами.
- **Рейтинги и отзывы:** Система оценок (1-5 звезд) и текстовых комментариев.
- **Ролевая модель:**
  - **Гость:** Только просмотр точек и отзывов.
  - **Пользователь:** Добавление новых точек, написание отзывов, редактирование своих записей.
  - **Администратор:** Полный контроль — удаление любых отзывов и точек.
- **SPA-эффект:** Быстрая подгрузка контента без полной перезагрузки страниц (Hotwire/Turbo).

##  Технологический стек

- **Backend:** Ruby 4.0.1, Ruby on Rails 8.1.2
- **Database:** PostgreSQL
- **Auth:** Devise
- **Frontend:** CSS (Custom Styles), HTML5 (ERB), Turbo Frames & Streams
- **Environment:** WSL (Ubuntu)

##  Права доступа (Roles)

В проекте реализована гибкая система разграничения прав доступа:

*   **Администратор:** Имеет полный доступ к управлению контентом. Может удалять любые точки и отзывы. 
    **Как получить:** Зарегистрироваться с почтой, указанной в `app/models/user.rb` (по умолчанию `admin@rnd.ru`), либо получить статус через `role: 1` в БД.
*   **Владелец:** Пользователь, добавивший точку. Имеет исключительное право на её редактирование и удаление (защищено логикой `authorize_owner!` в контроллере).
*   **Пользователь:** Авторизованный юзер. Может добавлять новые места и оставлять отзывы к любым точкам.
*   **Гость:** Неавторизованный посетитель. Доступен только просмотр списка шаурмечных и отзывов.

##  Структура проекта (Back-end часть)

Проект построен на классической архитектуре MVC. Основные компоненты бэкенда:

*   `app/models/` — Описание бизнес-сущностей, валидации и расчет средних рейтингов.
*   `app/controllers/` — Логика обработки запросов, фильтрация доступа и аутентификация.
*   `app/views/` — Шаблоны страниц. Использование **Partials** для форм и элементов интерфейса.
*   `db/migrate/` — История миграций и структура таблиц PostgreSQL.
*   `db/seeds.rb` — Скрипт для первичного наполнения базы реальными данными по Ростову-на-Дону.
